## 2024-03-17 - Avoid Static Empty Array Anti-Pattern in Inline Vue Handlers
**Learning:** Returning a static empty array (e.g. `const EMPTY_ARRAY = []`) to prevent render thrashing is completely ineffective if the parent function subsequently wraps it in methods like `.map()`, `.flatMap()`, or `.filter()`, because these inherently allocate a new array reference on every single invocation. Attempting to use a static constant here is a misleading micro-optimization that fails to solve the O(N) re-render problem.
**Action:** When an inline function call inside a `v-for` loop causes render thrashing due to creating new array references, explicitly extract the loop body into a dedicated child component. Within the child component, wrap the expensive mapping/filtering logic in a Vue `computed` property. This ensures the derived array reference is securely memoized and will only re-evaluate when its specific reactive dependencies change, isolating updates and preventing O(N) thrashing during rapid parent state changes (like LLM text streaming).
